Programación Orientada a Objetos

  • Es un modelo de programación que se basa en objetos para organizar el diseño de software. En este paradigma, los programadores se centran en los objetos que necesitan manipular, en lugar de en la lógica necesaria para hacerlo.
  • En la POO, los objetos tienen atributos que representan sus características o propiedades, y métodos que definen las acciones que pueden realizar. Las clases se utilizan para definir la estructura y el comportamiento de los objetos. Una clase actúa como una plantilla o un modelo a partir del cual se crean objetos individuales con características únicas.

Las clases en java

En Java, las clases son una parte fundamental de la POO y son la base para la creación de objetos, que son instancias de esas clases. Una clase en Java es un plano o un modelo que define la estructura y el comportamiento de los objetos que se pueden crear a partir de ella.

Uso de una clase en Java


						public class Persona {
							// Atributos
							private String nombre;
							private int edad;
						
							// Constructor
							public Persona(String nombre, int edad) {
								this.nombre = nombre;
								this.edad = edad;
							}
						
							// Método
							public void saludar() {
								System.out.println("Hola, soy " + nombre + " y tengo " + edad + " años.");
							}
						}

Conceptos fundamentales de las clases

Atributos

Las clases pueden tener atributos, también conocidos como variables de instancia, que representan las características o propiedades del objeto.


					public class Persona {
						// Atributos
						public String nombre;
						public int edad;
					}

Metodos

Son funciones propias de la clase que representan el comportamiento del objeto, pueden manipular los atributos, en el ejemplo de la clase Persona, saludar es una accion y refleja un comportamiento de esa persona. 


					public class Persona {
						// Atributos
						public String nombre = "Sebastian";
						public int edad = 18;
						// Método
						public void saludar() {
							System.out.println("Hola, soy " + nombre + " y tengo " + edad + " años.");
						}
					}

Constructor

El constructor es un metodo especial al cual nos sirve cuando estamos creando un objeto o instancia de nuestra clase, es util cuando queremos incializar ciertos atributos y hacer mas personalizable el comportamiento de nuestro objeto. 


					public class Persona {
						// Atributos
						private String nombre;
						private int edad;
					
						// Constructor
						public Persona(String nombre, int edad) {
							this.nombre = nombre;
							this.edad = edad;
						}
					}

Encapsulamiento

Es un principio de la POO que se refiere a la ocultación de los detalles de implementación de una clase y la exposición controlada de sus atributos y métodos. En Java, puedes utilizar modificadores de acceso como private, protected, public, etc., para controlar el acceso a los miembros de una clase.


					class Bancolombia {
						private String numeroCuenta;
						public String Nombre;
						public int edad;
						Bancolombia(String numeroCuenta, String Nombre,int edad) {
							this.Nombre = Nombre;
							this.edad = edad;
							this.numeroCuenta = numeroCuenta;
						}
					}

Herencia

Las clases pueden heredar atributos y métodos de otras clases. La herencia permite la creación de jerarquías de clases, donde una clase (subclase o clase derivada) puede extender o heredar características de otra clase (superclase o clase base).


					class Animal {

						String nombre;
					
						public void comer() {
							System.out.println("El animal está comiendo.");
						}
					}
					
					class Perro extends Animal {
					
						public void ladrar() {
							System.out.println("El perro está ladrando.");
						}
					}

Los modificadores de acceso

Los modificadores de acceso en Java son palabras clave que definen el nivel de acceso que tienen las clases, métodos y atributos desde otras partes del código. Sirven para controlar qué tan accesible es cada parte del código y proteger la información según sea necesario.

public

El modificador public indica que la clase, método o atributo es accesible desde cualquier parte del código, tanto dentro del mismo paquete como desde otros paquetes.


					public class Libro {
						public String titulo;

						// Constructor público
						public Libro(String titulo) {
							this.titulo = titulo;
						}

						// Método público para obtener el título del libro
						public String getTitulo() {
							return this.titulo;
						}
					}

Private

El modificador private restringe el acceso de la clase, método o atributo. Solo es accesible dentro de la misma clase. Es útil para proteger la información sensible.


					// Clase que representa una cuenta bancaria
					public class CuentaBancaria {
						// Atributo privado: solo accesible desde la clase CuentaBancaria
						private double saldo;

						// Constructor para inicializar el saldo
						public CuentaBancaria(double saldoInicial) {
							this.saldo = saldoInicial;
						}

						// Método público para consultar el saldo
						public double consultarSaldo() {
							return this.saldo;
						}

						// Método público para depositar dinero
						public void depositar(double cantidad) {
							this.saldo += cantidad;
						}
					}

Protected

El modificador protected permite que el miembro de la clase sea accesible desde clases del mismo paquete o desde clases que heredan de esta. Es útil en situaciones de herencia.


					// Clase que representa un empleado
					public class Empleado {
						// Atributo protegido: accesible solo desde la clase y sus hijas
						protected String nombre;

						// Constructor
						public Empleado(String nombre) {
							this.nombre = nombre;
						}

						// Método protegido: solo accesible en esta clase y sus subclases
						protected String obtenerNombre() {
							return this.nombre;
						}
					}

					// Clase hija que representa un gerente
					public class Gerente extends Empleado {
						
						public Gerente(String nombre) {
							super(nombre);
						}

						// Método para mostrar el nombre del gerente
						public void mostrarNombre() {
							System.out.println("Nombre del gerente: " + obtenerNombre());
						}
					}

Modificador de acceso por defecto (sin palabra clave)

Si no se especifica un modificador de acceso, se aplica el acceso por defecto, también llamado "paquete" o "friendly". Solo es accesible dentro del mismo paquete.


						// Clase que representa un producto en una tienda
						class Producto {
							// Atributo con acceso por defecto
							double precio;

							// Constructor
							Producto(double precio) {
								this.precio = precio;
							}

							// Método con acceso por defecto para consultar el precio
							double obtenerPrecio() {
								return this.precio;
							}
						}

Los Getters y Setters En Java

Son tipos de metodos que definen como se accede a una propiedad

Los Getters

Son metodos (funciones) los cuales solo pueden retornar un valor, osea que no pueden recibir argumentos como un metodo normal, pero siempre tienen que retornar algo, ese algo puede ser una propiedad dentro del objeto, como un getter que retorne un dato bancario en una clase banco. 


								class Bancolombia {
									private String numeroCuenta;
									public String Nombre;
									public int edad;
									Bancolombia(String numeroCuenta, String Nombre,int edad) {
										this.Nombre = Nombre;
										this.edad = edad;
										this.numeroCuenta = numeroCuenta;
									}
									public String obtenerNumeroDeCuenta() {
										return this.numeroCuenta;
									}
								}

Los Setters

Los setters son metodos especiales que solo son utiles a la hora de editar el valor de una variable dentro del objeto y no deberian retornar nada pero reciben solamente un parametro, el cual se va a convertir en el nuevo valor de la variable que queremos editar, son utiles pues podemos asegurarnos de que el valor ingresado cumpla ciertas reglas o especificaciones para esa propiedad. 

					
						class Bancolombia {
							private String numeroCuenta;
							public String Nombre;
							public int edad;
							Bancolombia(String numeroCuenta, String Nombre,int edad) {
								this.Nombre = Nombre;
								this.edad = edad;
								this.numeroCuenta = numeroCuenta;
							}
							public void colocarNombre(String nombre) {
								this.Nombre = nombre;
							}
						}
